# CIS320 Team Project Outline — ParkingFinder.com

## Project Overview

**ParkingFinder.com** is an AirBnB-style online marketplace connecting parking space landlords and tenants globally, handling listings, bookings, payments, and reviews.

---

## Week 1 (15 pts)

### 1. Use Cases (5 pts)
- Identify at least 5 use cases using the user goal technique or GenAI
- Provide a table with: User, User Goal, Use Case Name, Brief Description

### 2. Use Case Diagram (5 pts)
- UML diagram (e.g., MS Visio) showing actors and use cases

### 3. Non-Functional Requirements (5 pts)
- 10 requirements, 2 each for:
  - Usability
  - Reliability
  - Performance
  - Security
  - Design Constraints

---

## Week 2 (15 pts)

### 4. Domain Model Class Diagram (5 pts)
- UML Class Diagram with classes, attributes, associations, and cardinality

### 5. Book a Parking Space — Full Use Case + SSD (10 pts)
- Fully developed use case description
- System Sequence Diagram (SSD)

---

## Week 3 (35 pts)

### 6. Database Schema (5 pts)
- Tables with primary/foreign keys derived from the domain model
- Table format or ERD tool (e.g., MS Visio)

### 7. UI Design for "Book a Parking Space" (10 pts)
- Wireframes using MS Visio, a design tool, or GenAI

### 8. Spec Kit Spec-Driven Development (20 pts)
- Run Spec Kit to generate `spec.md`, `plan.md`, `tasks.md`
- Review generated files for accuracy
- Screen-record the generated user interface
- Submit all `.md` files and the screen recording

---
